11招玩转网络安全

0x00综述

这本书还有一个副标题呢，用Python，更安全～然而也没看到书里面提到了多少的Python。整本书干活寥寥无几，章节行篇布局隔的挺远，看起来倒是舒服得很，半天时间就看完了。与其说Python，不如说跟DVWA结合的挺紧密的，挺多例子都是拿DVWA说事。

  • 操作系统

    0x02 get到的点

  • nexpose这个扫描工具，还可以直接exploit-db，metasploit联动，这一点蛮让人惊艳的
  • 每次看到Burp里面Intruder模块payload的四种配置方式，总是一阵子头皮发麻，从这本学到这个点了
  • Burp的又一个新技能，提取response中返回的token直接在intruder中使用
  • zoomeye的链接总是记不住
  • mysql可以限定远程IP登陆失败次数
  • sqlmap设置代理、使用谷歌 python sqlmap.py -proxy http://127.0.0.1:1080 -g ‘inurl:’ –batch

0x04 小实验

  • Heart Bleed漏洞
  • struts2漏洞
  • tor匿名
  • 内网攻击 cain的使用
  • 无线网的密码-这个周末可以在实验室操作下😂

需要进一步加油的

  • sqlmap的功能特性图及tamper
  • nmap的功能特性图及script
  • hydra的各种参数 还没有试过这个呢（已经用过了～然而没有爆出密码）

最后

也不能说这本书不好，至少我看这本书没太多干货，学起来就没那么枯燥。然后又确实有挺多东西，是自己掌握的不够扎实。读这本书还是梳理清了挺多东西。所以嘛，这种事儿还是看匹配度，合适的就是好的了。感谢遇上～

dedecms5.5文件上传漏洞复现

0x00 背景

最近的从防火墙检测流量发现一大波webshell上传威胁告警，指向select_soft_post.php这个文件。查了一下发现是dedecms的一个任意文件上传漏洞，发生在5.5跟5.5final版本。

0x01 准备环境

  • 目前的环境
    之前一直习惯用xampp，虽然xampp比较稳定，但不支持多版本，更适合做开发及搭建版本稳定的学习环境。做漏洞复现，我们需要PHP运行环境的各种版本，相比而言，phpstudy才是更加适合目前操作的机器。
  • php dedecms5.5系统
  • 准备好的exp

0x02 动手搭建

下载phpstudy，解压安装到C盘根路径下，并开启，将dedecms5.5放到网站根路径

C:\phpStudy\PHPTutorial\WWW

解压获取的dedecmsv55系统放到该路径下，访问install,逐步完成安全，可见如下界面

至此，环境我们就准备好了，开始复现

0x03 漏洞利用

首先分析一下漏洞原因


变量$cfg_basedir没有正确的初始化，如果设置了global_register=on，我们可以通过覆盖变量的方式，饶过身份认证和系统变量初始化文件。

因此，这个漏洞的触发条件有两个，因为需要启动global_register,因此php的版本需要为5.2或者5.3，高版本的已经禁用了这个功能，这也是我们选择使用phpstudy来复现这个漏洞的原因。

  • dedecms5.5版本
  • php环境开启了global_register

接下来我们可以通告大佬写的一个表单上传，表单form对应的链接需要修改对应站点域名，然后来上传webshell文件。

对应的webshell,一句话后门

使用菜刀访问上传的webshell。

0x04 防御

从漏洞产生条件上来看，可以关闭global_register功能，当然其他的方法也很多，比如禁止使用select_soft_post页面

python之解放生产力

开篇

廖雪峰的教程，有一个很好的地方，顺着教程能够看清Python的全部面貌。

然而程序呢，并不是一个等所有知识点全学完了才可以动手写的事情。之前有过其他语言编程经验的人，往往在看过一遍语法的基础上就开始编程，只是简单的用法就能体现出”人生苦短，我用Python“的简洁高效，即使代码上依旧带有其他语言使用经验的编程风格。

虽如此，进一步的学习，还是有必要的。

人生苦短 我用Python

那么看看廖雪峰的教程提到了那些具有科技革命的高级特性。

切片

什么是切片呢，就是我们可以通过正负值的下标已经跳跃长度来截取变量。

逆置一个数组，曾经的写法可能是for循环中依此赋值，这个时候我们还要小心的处理下标，但是有了切片：
b = a[::-1]

注意一个要点，就是切片并不在原来数据上操作，

列表生成式

主要用于for循环等批量生成列表的地方。
我们最熟悉的写法可能是

for i in range(100):
    a.append(i)

使用列表生成式

a = [i for i in range(100)]

生成器

什么是迭代器：就是边循环便计算的机制。
生成器解决了一个问题，可以大批量生成的对象，随用随取，省了空间。
注意与列表生成式子的区别，

# 列表生成式
a = [i for i in range(10)]
# 生成器
b = (i for i in range(10))

上面的写法不需要初始化，可以直接使用next(g)来获取首位及下一位元素，更优雅的方式是使用for循环。

另一种更优雅的用法 yield

迭代

迭代强调了一个抽象度更高的for-in循环模式。list，tuple等都可以使用，然后如果像同步使用下标数字来引用元素的话，我们可以使用enumerate函数

for i, value in enumerate(['A', 'B', 'C']):
    print(i, value)

迭代器

写程序的时候经常遇到的一个错误：
object is not iterable。可以使用for-in情况的对象必须是可迭代的。

主要有两类：

  • 集合数据类型，如list、tuple、dict、set、str等；

  • generator，包括生成器和带yield的generator function。

那么，问题来了:如何判断是否对象可迭代？

from collections import Iterable
isinstance([], Iterable)

然而还有一个问题，可迭代与迭代器并不是一回事情。

Python的Iterator对象表示的是一个数据流，Iterator对象可以被next()函数调用并不断返回下一个数据，直到没有数据时抛出StopIteration错误。可以把这个数据流看做是一个有序序列，但我们却不能提前知道序列的长度，只能不断通过next()函数实现按需计算下一个数据，所以Iterator的计算是惰性的，只有在需要返回下一个数据时它才会计算。

Pyhton的高级函数编程

记得最早写Javascript的时候，接触到函数式编程，每种语言都有自己的一等公民，熟练使用一类公民才算得上是接触到语言的核心风格。

高阶函数

  • map/reduce
    惰性处理
  • filter 结合迭代器的无限数据流惰性运算，求素数使用的filter简直不能更完美。

  • sorted
    可以指定比较的键函数
    上面的函数从更加抽象，更加深层次的程度上优化了习惯的编程思维。总体来说，在需要的场合，通过对上面这些函数的使用能够替换原来臃肿的处理流程，并且逻辑上更加清晰，

  • 返回函数

既然是函数式编程，也就是说函数能够作为参数来使用，这边有一个很重要的概念式闭包。

  • 匿名函数 lambda

  • 装饰器

在函数之前或者之后执行某个操作，可以用来控制权限，写日志，绑定url和函数

  • 偏函数

关于python的一点实践

最开始写python，还是14年的时候开始。

很长一段时间里，都是写十几句的脚本。写一行代码，要百度三五次。一来是写的少，另一来是基础也就那样。不懂还不知道学习。
那个时候还在python challenge装逼。

最近在还是在写脚本，也没有写很多的东西，三五个文件，几百行代码。想把代码组织的清晰，就要有原则。

总结几条实践：

  • 命名是一个大问题，自己写的时候已经很注意了，然而还是有很多脸红的地方。
    • 函数使用小写 下划线分割
    • 类名使用首字母大写的驼峰方式
    • 文件名也要使用描述性的词汇，不要与使用的类库同名
    • 注意使用空行 类后两行分割
    • 默认参数中间的空格
  • 关于编程风格这一点，还是要多多琢磨pycharm的提示。消除提示的过程就是一个学习的过程。
  • import的机制还需要进一步加强理解
  • 马上把廖雪峰的python3教程看一遍，熟悉一些用法，来改进不优雅的地方。

既然在做着这个事情，就奋力打铁啊。
