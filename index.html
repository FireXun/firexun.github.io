mysql注释

注释可用于记录SQL语句的目的或存储过程中代码块的逻辑。在解析SQL代码时，MySQL会忽略注释部分。除了可执行注释之外，它只执行SQL部分。

MySQL支持三种注释样式：

单行注释

–单行注释。双破折号注释样式在第二个破折号后至少需要空格或控制字符（空格，制表符，换行符等）。

​ SELECT * FROM users; – This is a comment

请注意，标准SQL在第二个破折号后不需要空格。MySQL使用空格来避免某些SQL构造的问题，例如：

​ SELECT 10–1;

该语句返回11.如果MySQL没有使用空格，它将返回10。

‘#’单行注释。
SELECT​lastName, firstName`FROM
​ employees
WHERE
​ reportsTo = 1002; # get subordinates of Diane`

多行注释

C风格的评论/**/可以跨越多行。您可以使用此注释样式来记录SQL代码块。

/
​ Get sales rep employees
​ that reports to Anthony /

请注意，MySQL不支持嵌套注释。

可执行注释

MySQL提供可执行注释以支持不同数据库之间的可移植性。这些注释允许您嵌入仅在MySQL中执行但不在其他数据库中执行的SQL代码。

以下说明了可执行注释语法：

​ /! MySQL-specific code /
例如，以下语句使用可执行注释：
​ SELECT 1 /! +1 /
该语句返回2而不是1.但是，如果在其他数据库系统中执行它，它将返回1。

如果要从特定版本的MySQL执行注释，请使用以下语法：

/!##### MySQL-specific code /
字符串’#####’表示可以执行注释的MySQL的最低版本。第一个＃是主要版本，例如5或8.第二个2号（##）是次要版本。最后2个是补丁级别。

java反编译

一直使用jd-gui来的，不知怎么碰上了一个inner error，无法把源代码反编译出来。

然后在eclipse里面安装了插件，配置之后能够反编译了，但是还是不能一次获取所有源代码，无法在项目中查询需要的代码。

从https://varaneckas.com/jad/官网下载，然后使用命令
jad.exe -r -ff -d src -s java classes/*/.class

获取全部class反编译的java，然后进一步查看。

zvuldrill靶场

0x00 环境搭建

zvuldrill跟真实环境较为接近，难度也并不算高。作为DVWA入门之后练手的靶场，可以用来提高一下信心。

下载地址github link

具体本地环境搭建当然是选择PHPStudy，配置sys目录下config文件夹中密码。

使用自带的SqlFront建立数据库，导入sql文件，注意导入sql文件时候要选择utf-8文件格式。

完成环境搭建后，按照about页面所示，开启关闭相关模块。

0x01 修改lib.php文件

mysql_real_escape_string函数报错，无法正常留言。原因是调用该函数时候需要在调用前连接一下数据库，看代码发现lib.php文件之前在config.php有连接数据库，不清楚具体的原因，因此在这个地方我们直接把mysql_real_escape_string修改为addslashes来保证可以正常留言。

ios翻墙客户端

不折腾翻墙：

  • appstore里面的ssrConnectPro
  • 下载pp助手 安装shadowrocket

折腾下

周末，同学说起来可以自己编译一个shadowsocks-ios装在手机上。这样开始了好几天的折腾

0x00 下载代码

https://github.com/shadowsocks/shadowsocks-iOS.git

从github下载代码，然后切换git branch master切换分支到主分支，导入到xcode。或者直接利用xcode从github克隆。

0x01 修改description

Use of undeclared identifier ‘_description’; did you mean ‘description’?
解决方法：修改_description为_description_t，并在头文件加入

@property (nonatomic, strong, readonly) NSString *description_t;

0x02 openssl头文件

xcode从之前版本取消了对openssl的引用，所以需要自己去添加头文件查找路径。
尝试了很多brew link或者ln的方式都没有解决问题，最后是直接修改的xcode的build setting，增加对openssl的查找路径。
配置路径时候，注意如何选择编译环境。xcode里面的项目文件太多，不太清楚哪一个是主项目工程，所以选择编译路径的时候，要特别注意。

‘sodium.h’ file not found

$brew install cocoapods
$cd shadowsocks-ios
$pod init

修改podfile（注意~>符号，可以拷贝）

platform :ios, '9.2'
pod 'libsodium', '~> 1.0.3'

之后在项目路径下执行pod install，需要等很长一段时间。

11招玩转网络安全

0x00综述

这本书还有一个副标题呢，用Python，更安全～然而也没看到书里面提到了多少的Python。整本书干货寥寥无几，章节行篇布局隔的挺远，看起来倒是舒服得很，半天时间就看完了。与其说Python，不如说跟DVWA结合的挺紧密的，挺多例子都是拿DVWA说事。

0x01 两个工具命令分类

sqlmap参数说明
  • 选项说明
  • 目标说明
  • 请求
  • 注入 参数、–dbms说明数据库、–tamper、
  • 枚举 列举数据库信息、表结构和数据
  • 操作系统
nmap命令分类
  • 目标说明 包含哪些 排除哪些
  • 主机
  • 端口扫描技术
  • 端口说明与扫描顺序 默认1024
  • 服务和版本探测
  • 操作系统探测
  • 常见漏洞探测

    0x02 get到的点

  • nexpose这个扫描工具，还可以直接exploit-db，metasploit联动，这一点蛮让人惊艳的
  • 每次看到Burp里面Intruder模块payload的四种配置方式，总是一阵子头皮发麻，从这本学到这个点了
  • Burp的又一个新技能，提取response中返回的token直接在intruder中使用
  • zoomeye的链接总是记不住
  • mysql可以限定远程IP登陆失败次数
  • sqlmap设置代理、使用谷歌 python sqlmap.py -proxy http://127.0.0.1:1080 -g ‘inurl:’ –batch

0x04 小实验

  • Heart Bleed漏洞
  • struts2漏洞
  • tor匿名
  • 内网攻击 cain的使用
  • 无线网的密码-这个周末可以在实验室操作下😂

需要进一步加油的

  • sqlmap的功能特性图及tamper
  • nmap的功能特性图及script
  • hydra的各种参数 还没有试过这个呢（已经用过了～然而没有爆出密码）

最后

也不能说这本书不好，至少我看这本书没太多干货，学起来就没那么枯燥。然后又确实有挺多东西，是自己掌握的不够扎实。读这本书还是梳理清了挺多东西。所以嘛，这种事儿还是看匹配度，合适的就是好的了。感谢遇上～
