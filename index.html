人生苦短 我用Python

切片

什么是切片呢，就是我们可以通过正负值的下标已经跳跃长度来截取变量。

逆置一个数组，曾经的写法可能是for循环中依此赋值，这个时候我们还要小心的处理下标，但是有了切片：
b = a[::-1]

注意一个要点，就是切片并不在原来数据上操作，

列表生成式

主要用于for循环等批量生成列表的地方。
我们最熟悉的写法可能是

for i in range(100):
    a.append(i)

使用列表生成式

a = [i for i in range(100)]

生成器

什么是迭代器：就是边循环便计算的机制。
生成器解决了一个问题，可以大批量生成的对象，随用随取，省了空间。
注意与列表生成式子的区别，

# 列表生成式
a = [i for i in range(10)]
# 生成器
b = (i for i in range(10))

上面的写法不需要初始化，可以直接使用next(g)来获取首位及下一位元素，更优雅的方式是使用for循环。

另一种更优雅的用法 yield

迭代

迭代强调了一个抽象度更高的for-in循环模式。list，tuple等都可以使用，然后如果像同步使用下标数字来引用元素的话，我们可以使用enumerate函数

for i, value in enumerate(['A', 'B', 'C']):
    print(i, value)

迭代器

写程序的时候经常遇到的一个错误：
object is not iterable。可以使用for-in情况的对象必须是可迭代的。

主要有两类：

  • 集合数据类型，如list、tuple、dict、set、str等；

  • generator，包括生成器和带yield的generator function。

那么，问题来了:如何判断是否对象可迭代？

from collections import Iterable
isinstance([], Iterable)

然而还有一个问题，可迭代与迭代器并不是一回事情。

Python的Iterator对象表示的是一个数据流，Iterator对象可以被next()函数调用并不断返回下一个数据，直到没有数据时抛出StopIteration错误。可以把这个数据流看做是一个有序序列，但我们却不能提前知道序列的长度，只能不断通过next()函数实现按需计算下一个数据，所以Iterator的计算是惰性的，只有在需要返回下一个数据时它才会计算。

关于python的一点实践

最开始写python，还是14年的时候开始。

很长一段时间里，都是写十几句的脚本。写一行代码，要百度三五次。一来是写的少，另一来是基础也就那样。不懂还不知道学习。
那个时候还在python challenge装逼。

最近在还是在写脚本，也没有写很多的东西，三五个文件，几百行代码。想把代码组织的清晰，就要有原则。

总结几条实践：

  • 命名是一个大问题，自己写的时候已经很注意了，然而还是有很多脸红的地方。
    • 函数使用小写 下划线分割
    • 类名使用首字母大写的驼峰方式
    • 文件名也要使用描述性的词汇，不要与使用的类库同名
    • 注意使用空行 类后两行分割
    • 默认参数中间的空格
  • 关于编程风格这一点，还是要多多琢磨pycharm的提示。消除提示的过程就是一个学习的过程。
  • import的机制还需要进一步加强理解
  • 马上把廖雪峰的python3教程看一遍，熟悉一些用法，来改进不优雅的地方。

既然在做着这个事情，就奋力打铁啊。

大数据及机器学习，都是当前最火热的领域。周围很多同学都投入极大的激情，但是这个热情来的快，似乎去的也快。去年就又有一点感觉，似乎很多同学的基础并不够扎实。但是老师们往往默认基础已经足够好了。所以这里面更需要投入更多的时间，还是工作量问题吧。同时再一个要自己学习。

模块作用

模块 作用
Kafka 是一种高吞吐量的分布式发布订阅消息系统
zookeeper 一个为分布式应用提供一致性服务的软件，提供的功能包括：配置维护、域名服务、分布式同步、组服务等。

步骤说明

  • CentOS-7-x86_64-Minimal-1511.iso

安装所使用的系统镜像，注意到有mimimal，只包含必要的软件，很多软件被精简掉了，能够减轻占用资源，提高性能。

  • 关于防火墙和selinux

如果开启了防火墙，外部其他机器ping本机，在某些情况下，将会得不到响应，为了避免这种情况，我们有必要关闭防火墙。

selinux是一款为了提高系统安全性的软件：对系统服务，文件权限，网络端口访问有极其严格的限制。基本上安装完linux，惯例是关闭改功能。所以后面有一些端口或者即使在root权限下都不能将服务开启问题，可以考虑selinux是否正常关闭。可以在root账户下使用getenforce命令来确认。

  • 为什么我们需要两张网卡

实际意义上搭建集群，每个主机我们需要一张网卡就够了，用主机的这一张网卡，配置上合适的网关、子网掩码、IP。如此就能实现集群了。
我们的enp0s3网卡就是起到这个作用。同时使用nat方式，nat就可以连接外网了。局域网内集群的相互通信都是使用的该Ip地址。

另一张网卡我们是使用的host-only方式，该网卡的配置同时需要关闭dhcp功能，实际上该网卡唯一的作用
就是方便我们来使用真实的物理主机来连接虚拟机，或者说该网卡的作用就是方便我们来往虚拟机拷贝文件，然后使用终端工具来连接两个作用。

  • host文件

为每一个主机ip起了一个别名，大家可以注意到三个主机ip都是nat网卡地址，后面连接cluster1跟直接输入cluster1的ip效果是一样的。

  • 关于root用户、hadoop账户

咱们所用的软件基本上都是运行在非root账户、而是hadoop账户下。
在创建完文件之后，总有一个步骤是chown 来改变新建立文件的属主。这样使用hadoop账户运行所需要程序时候，才能对使用文件有读取权限。
注意：账户切换有时候挺麻烦的，一不小心就会在root账户下执行了，这个时候可能会在hadoop_files下面产生为root账户的一些日志文件。所以切换到hadoop后可能没有权限读取在root账户下产生的那些文件，这个时候不妨运行一下chown命令。

  • /etc/profile下的各种环境变量

我们在终端敲入的指令并不是黑魔法，都是有一个对应的文件。环境变量的作用就是告诉我们去那一个路径下面找输入指令所对应的文件。一般来说我们执行一个指令，会先在当前路径、个人path路径，系统path路径寻找。因此我们所添加的各种环境变量没有什么稀奇的，就是要执行软件所在的绝对路径。如果类似%%，表示引用了这个环境变量，这种可以看作是相对路径。

  • screen

一般我们敲完指令，会在终端启动起来任务占用当前的终端窗口，一旦我们关闭窗口。这个任务也被终止了，screen就是起到这个作用。退出窗口时候，不会结束任务。
screen -ls 列出当前窗口 -d 释放 -r 回复 -S 起一个名字

第五步

上面的代码因为cookie的值是硬编码在脚本中，这样一旦session过期，后面将会跳转到未登录装填。这就需要我们重新去更新cookie对应的值，那么有没有方法可以解决这个问题呢？
当然是有的。看一下requests提供的session解决方案，如下图

因此可以使用session来改造代码，requests中的session将会自动保存相关的cookie信息，如此每次运行就可以自动使用账号及密码登陆。
版本5代码

#coding:utf-8  
import requests
from bs4 import BeautifulSoup

s = requests.Session()
url_home = "http://www.home.com/"
s.get(url_home)

url_login = "http://www.login.htm"
data_login = {'account':'username','password':'passwd','rememberme':'on'}
text = s.post(url_login,data=data_login)

r = s.get(url_home)
soup = BeautifulSoup(r.text, 'html.parser')
random = soup.form.input.attrs["value"] 
content = soup.select_one(".content").contents[0]

mood_create = 'http://www.moodcreate.htm'
payload = {'radom': radom, 'content': content}
s.post(mood_create, cookies=cookies, data=payload)
