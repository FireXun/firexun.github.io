折腾下

周末，同学说起来可以自己编译一个shadowsocks-ios装在手机上。这样开始了好几天的折腾

0x00 下载代码

https://github.com/shadowsocks/shadowsocks-iOS.git

从github下载代码，然后切换git branch master切换分支到主分支，导入到xcode。或者直接利用xcode从github克隆。

0x01 修改description

Use of undeclared identifier ‘_description’; did you mean ‘description’?
解决方法：修改_description为_description_t，并在头文件加入

@property (nonatomic, strong, readonly) NSString *description_t;

0x02 openssl头文件

xcode从之前版本取消了对openssl的引用，所以需要自己去添加头文件查找路径。
尝试了很多brew link或者ln的方式都没有解决问题，最后是直接修改的xcode的build setting，增加对openssl的查找路径。
配置路径时候，注意如何选择编译环境。xcode里面的项目文件太多，不太清楚哪一个是主项目工程，所以选择编译路径的时候，要特别注意。

‘sodium.h’ file not found

$brew install cocoapods
$cd shadowsocks-ios
$pod init

修改podfile（注意~>符号，可以拷贝）

platform :ios, '9.2'
pod 'libsodium', '~> 1.0.3'

之后在项目路径下执行pod install，需要等很长一段时间。
