Web扫描器

《白帽子讲web扫描》
刚好也是最近在忙的毕业设计，多考虑一点，多写一点。
扫描器扫的是什么？是怎么进行的？
获取扫描目标，也就是带有参数的各种链接。
扫描目标从哪里来，信息收集，指纹识别，具体的指纹也并不是从空而来，而是一个的规则，能够使用机器识别的方式么？很难，缺乏统一的数据集。
目前主要是爬虫技术，然后是漏洞审计，根据规则来判定漏洞存在与否。
一款web漏洞扫描器应该具有那些功能：

  • web爬虫
    想办法爬取全部链接，不考虑这个问题，尽快设计url处理的模块及方式。
    重点在哪：
    url相似，包含、重复等问题
  • 扫描器具体的数据结构，软件架构、各功能的实现
    • 端口扫描与破解
    • 子域名收集
    • 敏感信息手机、指纹识别
    • 通用漏洞审计（sql注入，xss，命令执行，文件包含，敏感文件泄漏）
    • nday/0day漏洞审计（discuz dedecms heartbleed php 远程ddos）
