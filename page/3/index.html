windows下脚本配置IP地址

带着笔记本有时候在固定的地方工作，需要用到同一个的Ip地址。换个地方换个Ip，又要重新输一遍。

感觉这个过程很繁琐，因为是window工作环境，一开始想到了vbs脚本。

无意中发现了强大的netsh命令。。。。

下面分两个部分，先介绍netsh的基本用法，然后贴段vbs脚本实现自动配置IP地址功能。

netsh常见用法

查看网络配置 netsh interface ip show {选项}
配置接口IP/网关IP netsh interface ip set address “本地连接” static ip地址 子网掩码 网关
配置自动换取IP地址 DNS地址及 wins地址

netsh interface ip set address "本地连接" dhcp
netsh interface ip set dns "本地连接" dhcp
netsh interface ip set wins "本地连接" dhcp

配置静态IP地址 DNS地址 及 wins地址

netsh interface ip set address "本地连接" static ip地址
netsh interface ip set dns "本地连接" static 子网掩码
netsh interface ip set wins "本地连接" static 网关

查看网络配置文件 netsh -c interface dump 使用管道 > 可导出为网络配置文件
导入网络配置文件 netsh -f 文件位置 或者 netsh exec 文件位置

laravel项目迁移

最近接触了一个使用larval框架的php项目，虽然有过一些tp5开发的经验，对于环境的认识也还是很不专业，见招拆招了。

服务器环境是linux系统，项目是从别人的手上半途接过来，因此首要的任务就是熟悉环境，并在本地搭建。

登陆服务器

开启vpn，使用个人域名账号登陆跳板机，然后通过跳板机跳转到测试服务器。

下载相关源代码

在文件路径下直接使用将项目打包，根据之前的文档了解到可使用sz，rz指令来实现文件的上传下载，具体两个指令使用如何配置在此处不再赘言。

数据库配置

根据文档及代码确定mysql数据库所在机器，导出相关数据库并且合理配置。

过程中发现，数据库ip没有使用配置文件，项目中统共使用了四个不同ip地址上的数据库，具体地址分散于各个文件中，不方便使用，后期可重构解决该问题。

本地win10环境

我直接使用的xampp集成工具来配置开发环境。但是将文件下载后发现，无法正常运行。
修改了laravel框架下config/app.php中debug模式，发现上一步的打包未曾包含.env等隐藏环境配置文件。

若要包含隐藏文件，可使用一下两种方式

  • tar -rf
  • 直接打包所有目录

配置虚拟域名

至此，项目预览基本没有问题。测试中发现部分链接引用的相对路径出错。因此可以配置xmapp文件，为当前项目开启虚拟路径。
具体方式如下
修改apace/conf/extra/httpd-vhosts.conf
添加如下代码

<VirtualHost laravel.dev:80>
    DocumentRoot "D:\xampp\htdocs\laravel\public" 
    ServerAdmin laravel.dev
      <Directory "D:\xampp\htdocs\laravel">
        Options Indexes FollowSymLinks
        AllowOverride All
         Require all granted
      </Directory>
</VirtualHost>

注意开启nameVirtualHost*:80 这一行。
至此，环境搭建完全完成。

编程入门路(数据结构+算法)

之前是在小地方，总觉得自己水平足够了，其实还差的远呢。以前觉得学好C语言毕业的时候就可以呼风唤雨了。想在才觉得，不入流学校的高配还不如大城市的低配。

之前写了C语言的文章，这个只能说是大一半年的目标。一个学期，能过了语言关，对代码有了悟性，才算是半只脚进了编程的大门了。后面还有很多路要走。

接下来咱们说说语言熟练之后的路途。

大神说程序=数据结构+算法。

那么数据结构有哪些东西，我们需要做到什么程度？如果说C语言能靠40-60道练习题目就学好，数据结构有没有这么一个指标？

我说一点，就是手写算法。手写算法就是一种刻意练习，计算机的好处就是可以随时动手，不动手只看书，脑子里面就只是模糊的印象。

数据结构：

  • 推荐的还是严蔚敏 吴伟民老师的练习册
  • 基本的栈，队列。我们现在很少讲怎么去实现栈、队列了。在这里要记住、栈、队列解决问题的几个典型场景，这些场景的算法一定能够不依赖任何资料，完全手写出来。
  • 树 二叉树（递归）
  • 图 搜索算法 （最小二叉树的生成，最短路径的生成）
  • 然后是排序算法 （一定要手写一遍，然后考虑优化）

上面提到的过一遍，最好的阶段当然是大一下学期这半年的时间，其次就是现在。

算法

  • Master定理 分析推理 复杂形式的算法复杂度
  • 暴力 （这种思想是大家比较容易接受的）
  • 分治
  • 动态规划
  • 还是算法跟典型使用场景的结合，每种要考虑6-10次典型练习。
  • 再次强调一点，一定要手写算法。

做这些，最好的时候是大二上学期，其次是现在。

我的C语言入门经历

0x00前言

距离第一次接触C语言编程，过去很长时间了。这中间陆续学习了很多门语言，零散的写过一些代码，然而水平并不高。在C语言上，我的能力也大概就是能够独立完成课后设计，数据结构相关的题目，二叉树、图等的课后设计，可以使用C或者C++独立实现。后面写的Java Web代码稍多一点。做过的代码行数最多的也就是在linux下实现过一个基于控制台的局域网通信项目。

在C语言上，因为做过的很多项目都是一些课后作业性质的，缺乏大项目洗礼、锻炼。只能点到为止，希望对于感觉C语言学习有困难有所帮助，并同时期望阅读到本文的新生们能够在大二结束时候达到独立完成各种使用C语言的作业的水平，对于VS等IDE也有较深程度的了解。

0x01 忘不了的黄昏

我大学时候使用的教程是钱能的C++教程，一本红皮书。记忆里，那是个带着晴朗空气的九月下午，从学校科技馆提前一节下课，到了信息楼实验室。

因为没有带课本，就在学校的老机器上凭记忆码着hello world。写完后运行了一下，有几个错误，然后定位到错误修改。再次运行，0 error 0 warning。真是最美的初体验，或许是占了记忆力稍好一点的优势，从hello world中得来了自信心。下机后，拿到课本在读一边关于hello world的介绍，从include开始，到最后的return，一点一点去理解行行代码的实际意思，恍然大悟，编程的感觉很好哇。

实际上现在是10月份下旬了，大家肯定都学完这个经典的入门程序了。如果要你纸上默写一下这个程序，能不能默写的出来？能不能讲出每一行代码的作用？。这就是第一步，千里之行，始于hello world。Hello world虽小，五脏俱全。关于编程，关于计算机体系，关于API，都在这小小的几行代码了。

0x02 初窥编程门径

实际上这个不应该算是第二阶段，算作1.5阶段比较合适。
这个节点是怎么样的呢？
学完for、while、do while循环，很多人还是写不出代码，觉得看是看的懂了，就是写不出，通病。那我们应该怎么解决？

我是通过打印第一个平行四边形起家的。想做得出这个没别的办法，先看书上例子。看完例子，做习题，琢磨哪一些是在变化的量，哪一些没有在变化。替换掉那些行数，列数的数字，能够独立成功打印出一个三角形，一个平行四边形，对于循环等结构的具体执行过程也就聊熟于心了。

实际上过了上一关，编程就没有问题了。剩下的就是学习各种语言相关的细节了，C语言的话，就是字符串、指针、结构体等知识，后面复杂点有位运算、共用体、枚举结构。加上最后的文件操作。有了前面的例子，后面认真听课，完成老师的作业，抄一抄课本上样例代码，就结束了。真的，C语言学起来就应该是如此轻松。因为当独立打印出平行四边形时候，你就已经领悟了编程的本质。

0x04 练习加强基础

在我的C语言学习经历中，再新的阶段就是大一的寒假了。寒假在家，也没有笔记本电脑。还是在纸上将书本上的所有代码抄写了一边。

其实我个人资质是很平庸的，写代码到今天，也没有取得很大的成绩，但是至少在写代码这条道路上找到了一点儿方向感，希望这微薄的话语能够给大家带来一点前行的力量。在开始的时候遇到一些困难是应该的，觉得时间被浪费很多也是必然的，一定要有一个练习的心态。

简版-数语以敝之

秘诀

一本语法书+50道练习题=入门

达成目标

  • 独立完成C语言的课程设计，看懂几百行甚至千行内的代码（能力水平判定）
  • 通过对C语言的学习加强计算机修养（软性）

简短版-C语言升级打怪之路

  • 第一关：手写hello world调试运行，理解每一行代码
  • 第二关：使用顺序、判断、循环三种结构打印规律的复杂图形
  • 第三关：认真阅读C语言语法书上的其他内容
  • 第四关：50道题目左右的编程训练
  • 第五关：独立完成课程的大作业
